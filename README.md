# Трекер питания

Приложение для отслеживания потребления калорий и макронутриентов с поддержкой библиотеки продуктов, дневника питания, статистики и экспорта данных.

## Особенности

- 📅 **Навигация по дням**: Просмотр и редактирование дней текущей недели
- 🍎 **Библиотека продуктов**: Единая база продуктов с КБЖУ на 100г
- 🏷️ **Штрихкоды**: Поддержка сканирования штрихкодов (заглушка API)
- 📊 **Статистика**: Графики и прогресс-бары по калориям и макронутриентам
- 🥗 **Группы продуктов**: До 3 уровней вложенности для организации приёмов пищи
- 📤 **Экспорт/Импорт**: Сохранение и загрузка данных в формате JSON
- ⚙️ **Настройки**: Целевые значения КБЖУ и настройки приёмов пищи

## Технологии

- React 18 + TypeScript
- Vite (сборка)
- Tailwind CSS v4 (стили)
- dnd-kit (drag-and-drop)
- Recharts (графики)
- Lucide React (иконки)
- localStorage (сохранение данных)

## Установка и запуск

### Предварительные требования

- Node.js 18+ и npm

### Установка

1. Клонируйте репозиторий:

2. Установите зависимости:
   ```bash
   npm install
   ```

3. Запустите приложение в режиме разработки:
   ```bash
   npm run dev
   ```

4. Откройте браузер по адресу [http://localhost:3000](http://localhost:3000)

```bash
npm run build
```

Собранные файлы будут находиться в папке `dist`.

## Использование

### Добавление продуктов

1. **Через библиотеку**: Нажмите "Добавить блюдо" в любом приёме пищи, затем выберите продукт из библиотеки или добавьте новый.
2. **Вручную**: Введите название и КБЖУ на 100г.
3. **По штрихкоду**: Нажмите "Сканировать" в библиотеке (реализована заглушка).

### Работа с дневником

- **Выбор дня**: Нажмите на кружок с датой в верхней панели
- **Добавление приёма пищи**: Настройте приёмы пищи в настройках
- **Группировка**: Создавайте группы для организации блюд (до 3 уровней вложенности)
- **Изменение веса**: Нажмите на иконку карандаша рядом с весом порции

### Статистика

- **Дневная статистика**: Круговая диаграмма распределения калорий и прогресс-бары
- **Статистика за период**: Графики и средние/суммарные значения за выбранный период
- **Цели**: Сравнение с целевыми значениями из настроек

### Экспорт и импорт

- **Экспорт**: Выберите период и скачайте JSON-файл со всеми данными
- **Импорт**: Загрузите JSON-файл для восстановления данных

### Настройки

- **Целевые КБЖУ**: Установите целевые значения калорий и макронутриентов
- **Приёмы пищи**: Настройте названия и количество приёмов пищи по умолчанию

## Архитектура данных

Приложение использует две основные сущности:

1. **LibraryItem**: Продукт в библиотеке с КБЖУ на 100г
2. **DayRecord**: Запись дня с приёмами пищи

Связь осуществляется через `libraryItemId` в `FoodLogItem`. При изменении продукта в библиотеке автоматически пересчитываются все связанные записи.

## Граничные случаи

- **Удаление продукта**: Продукт не удаляется, а архивируется, если есть ссылки в истории
- **Коллизии имён**: При дублировании названий отображается короткий ID
- **Ограничение вложенности**: Максимум 3 уровня вложенности групп
- **Частичный импорт**: Импортируются все необходимые продукты из библиотеки
